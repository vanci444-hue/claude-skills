# 笔记风格详解（带 XML 示例）

样板：`https://istir4q565.feishu.cn/wiki/W5COw79dBiwKWekAXGRcyOionyh`

这是从 Evan 的雅慧课程笔记里反推出的写法规律。所有新整理的笔记都按这套来。

---

## 一、整体骨架

```
H1 大模块（课程的一"幕"）
  H2 主题
    H3 / H4 细分点
    p / ul / 表格 / 代码块 / callout / 画板
  ── hr ──
H1 下一大模块
  ...
```

- **H1 之间、大 H2 之间用 `<hr/>` 分隔**，让文档有清晰的"章节呼吸感"。
- 标题层级跟着课程的逻辑下钻，不要为了好看强行加层级。
- 高价值章节标题后缀 **🌟**（样板里"数据飞轮 🌟""核心能力全景图 🌟"）。

---

## 二、每种块的判定与写法

### 1. 段落 p — 概念定义 / 结论 / 过渡
用于一句话定义、单点结论、承上启下。常以加粗术语开头。
```xml
<p><b>AI产品：</b>通过人工智能技术实现自主决策或智能服务能力的数字化产品形态，其本质是数据驱动的价值创造系统。</p>
```

### 2. 列表 ul/li — 并列要点（加粗引导）
特征、能力项、清单。**每条用 `<b>关键词：</b>` 引导**，后接解释。可嵌套子列表。
```xml
<ul>
  <li><b>持续进化</b><ul><li>模型一直在进化</li><li>随数据和反馈变强</li></ul></li>
  <li><b>动态适应</b><ul><li>按上下文、Memory 及时调整</li></ul></li>
</ul>
```
⚠️ 必须用原生 `<ul><li>`，**禁止**在 `<p>` 里手敲 "• / · / -"。

### 3. 表格 — 真·矩阵
判据：**每行字段结构一致的多属性对比 / 枚举**。样板里的表格场景：
- 维度对比（传统产品 vs AI 产品）
- 阶段 → 角色 / 阶段 → 流程节点（2 列映射）
- L1–L5 分级（多列：层级/阶段/描述/技术/商业模式/案例/价值/时间）
- 薪资职级（薪资/年限/负责范围）
- 枚举清单（公众号：序号/名称/简介；生产力工具：序号/场景/工具/用法）

```xml
<table>
  <colgroup><col width="100"/><col width="190"/><col width="305"/></colgroup>
  <thead><tr><th><b>维度</b></th><th><b>传统产品</b></th><th><b>AI产品</b></th></tr></thead>
  <tbody>
    <tr><td>核心驱动力</td><td>预设功能逻辑</td><td>数据 + 算法迭代</td></tr>
  </tbody>
</table>
```
⚠️ **任何单元格都不加 `background-color`。** 表头只用 `<b>`。

### 4. 代码块 pre — 原始素材 / SOP 清单 / 面试答案
判据（满足其一就用代码块，不用表格也不用 callout）：
- **要逐字保留的原始素材**：如一条 JD 原文。
- **带编号层级的 SOP / 清单**：`01 …` / `1.1 …` 这种多级编号、顺序性强的内容。
- **面试题的"标准答案"要点**：方便整段背 / 抄。

`<pre>` 配 `caption` 给它命名；多行用 `<br/>`。
```xml
<pre caption="提算法需求（0-1）" lang="Plain Text"><code>01 明确业务背景及用户使用场景<br/>1.1 为什么做这件事，告诉算法产品价值<br/>1.2 什么人在什么场景下怎么用，期望达到什么目的<br/><br/>02 明确算法的输入输出<br/>...</code></pre>
```
样板里的代码块实例：「一个好的 JD」(JD 原文)、「认知极限」(概念陈述)、「提算法需求 0-1」、「算法效果跟踪 1-100」。

### 5. callout — 术语卡 / 提示
- **📖 术语 / 知识点 / 概念占位**：把一个要单独高亮的概念拎出来。样板：解决方案、产品运营、FDE。
- **💡 提示 / 经验 / 避坑**：操作建议、踩坑提醒。样板：养龙虾做自动化订阅、友情提示。

```xml
<callout emoji="📖" background-color="rgb(245,246,247)" border-color="rgb(254,212,164)">
  <p><b>B端业务"解决方案"，又称售前 / GTM</b></p>
</callout>
```
callout 允许用背景色（这是表格禁色规则的例外）；沿用样板配色 `background-color="rgb(245,246,247)" border-color="rgb(254,212,164)"`。

### 6. 画板 whiteboard — 空间 / 关系结构
判据：内容**天然是图**——架构分层、流程、脑图、泳道、关系网、时间线、带结构的对比。
→ 调用 `/feishu-whiteboard-draw`，产出**原生可编辑节点**（框字一体），不是 SVG 死图。
样板里的画板：产业生态图、三层架构、工作流、核心能力全景图、数据飞轮、找 AI 节点、概念验证、面试准备、大厂vs小厂、新模型时间线。

### 7. bookmark / img
- 外链用 `<bookmark name="标题" href="..."></bookmark>`。
- 课件原图（不值得重画成画板的）直接贴 `<img>`。

## 三、术语保留（不翻译）

ASR 转写常把英文术语写成谐音错字，**还原成正确英文，且不译成中文**：
query、intent、bad case、slot、EDD（Evaluation Driven Development）、Agent、RAG、MCP、Function Calling、Workflow、Skill、Human-in-the-loop（人在环路）、Fine-tuning、MaaS、ROI、PRD、JD、Demo、MVP、CoT、RLHF…

常见 ASR 谐音还原例：「卡瑞 / 花瑞」→ query，「阜阳利」→ bad case，「曹魏 / 仓位」→ slot，「阿里云（指能力）」→ 多为 "Agent / Harness" 语境，「欧扑斯」→ Opus，「Germany」→ Gemini，「纳米 banana」→ Nano Banana。结合上下文判断。

## 四、内容深度的"刻度"

- **删到只剩骨头**：一节 1500 行的 ASR，最终笔记可能就十几个块。口语占比极高，留存率低是正常的。
- **保留"可考 / 可用"的硬货**：定义、对比、SOP、面试题答案、岗位判断标准、避坑、工具清单、模型/公司名录。
- **例子要留但要压缩**：雅慧爱用生活化例子（点奶茶、道歉信、客服转人工），保留 1 个最典型的，删掉重复铺垫。
- **对照截图补准确措辞**：表格列名、架构层级名、L1-L5 这类，以截图 / 课件为准，ASR 口述为辅。
